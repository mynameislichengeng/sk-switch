# UI 设计原则

记录这个项目里 TUI 视觉与交互层面的统一约定。新增功能或修改现有界面时遵循这些原则，避免风格漂移。

## 颜色

### 弹窗（popup）一律使用橘红色

所有 popup 的边框、标题色统一使用同一组橘红色：

| 主题 | 色值 |
|------|------|
| 浅色 | `#B22222` |
| 深色 | `#FFA07A` |

**实现**：`tui/styles.go` 中定义了 `popupBorderColor`，所有弹窗必须通过它来设置边框/重点色，不要再写颜色字面量。

适用范围（当前）：
- 数据源 / agent 配置 — 新增 / 编辑 / 删除三种弹窗
- 技能列表 — 「技能详情 / 分配」弹窗

理由：
- 弹窗是临时性、阻断性的浮层，颜色统一让用户对"这是个对话框"形成肌肉记忆
- 跟 tab 选中态的绿色背景拉开区分，避免视觉混淆

### 选中 / 聚焦态（绿色）

绿色 `#006400 / #90EE90`（亮 / 暗主题）用于"当前选中 / 聚焦 / 命中"语义。具体两种用法：

| 场景 | 表现 | 例子 |
|------|------|------|
| 选中的容器 | 绿色背景 + 白色加粗前景 | `tab` 当前页（背景 `#006400/#228B22`） |
| 重点 / 聚焦的文本 | 绿色加粗前景 | 表头文字、弹窗内当前编辑字段、已分配状态 `[ Y ]` |

行光标用 `lipgloss.Reverse(true)`（终端反色），不依赖颜色，避免在不同主题下色差异常。

### 错误信息（红色）

`❌` 错误信息统一使用 `#FF6B6B`，仅在弹窗内显示。**关闭弹窗后**，错误状态立刻清除，不要在主列表残留。

## 布局

### 弹窗位置

- 水平：居中
- 垂直：靠上 1/4 处（`(height - boxHeight) / 4` 行 padding）

不使用完全垂直居中，目的是给下半屏留呼吸空间，让信息层级更清晰。

实现：`ConfigListModel.placePopup` / `ListModel.renderAssignPopup` 末尾的 `lipgloss.Place(... Center, Top, ...)` + 手动顶部 padding。

### 行间距

列表中每条数据行之间留 1 行空行；列表与帮助行之间留 2~3 行空行。终端是字符网格，靠空行制造透气感。

### 弹窗内边距

`lipgloss.Padding(2, 5)`（上下 2 / 左右 5）作为带表单 / 多字段弹窗的标准。简单确认弹窗可以用 `Padding(1, 3)`。

## 交互

### 弹窗按键约定

| 按键 | 行为 |
|------|------|
| `Tab` | 切换字段焦点 |
| `Enter` | 确认/提交 |
| `Esc` | 取消/关闭，不保存 |
| `↑/↓` 或 `j/k` | 列表内移动 |
| `空格` | 切换状态（如 toggle 分配） |

不要使用 `y / n` 作为确认快捷键 —— 已统一改成 Enter / Esc。

### 错误提示

- 校验失败 → 错误显示在弹窗内（红色 `❌`），保持弹窗不关
- 用户改完后再次 Enter 触发重新校验
- 关闭弹窗时清空错误，避免污染下次打开

### 帮助行

底部帮助行只列**当前视图能用的全局快捷键**，不要堆全部按键。例如技能列表只显示：
```
r 刷新 | Tab 切换 | q 退出
```
列表内的上下移动、空格弹窗等约定俗成的操作不必赘述。

## 文案

- 单位：避免"个"等口语化单位词。例如 `总数：36` 而非 `总数：36 个`
- 标签：选用动词中性词。例如 `开启 / 关闭` 而非 `显示 / 隐藏`
- 提示语：保持动作-对象简洁，如 `Esc 取消` 而非 `按 Esc 键取消操作`
