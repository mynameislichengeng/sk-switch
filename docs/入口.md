# 入口 · 选择模块

启动 sk-switch 时（首次或 `last_module` 缺失）会弹出全屏"选择模块"弹窗，让用户决定本次进入哪个模块；之后会自动记住，下次启动直接进入。

## 三个模块

| 选项 | 说明 |
|------|------|
| **SKILLS** | 管理 skill / 数据源 / agent 分配 |
| **MCP** | 管理 MCP server / agent 分配 |
| **SETTINGS** | 主题色、运行配置 |

## 状态机

```
                   first_run=true (no last_module)
                         │
                         ▼
                  ┌──────────────┐
        ┌────────▶│  入口弹窗     │
        │         └──────────────┘
        │ Ctrl+P         │ Enter
        │                ▼
        │        ┌────────────────┐
        │        │  module + tab  │  Tab/Shift+Tab 在模块内切 tab
        └────────│  内容渲染      │  写入 last_module → 下次直接进
                 └────────────────┘
```

## 快捷键

入口弹窗内：

| 按键 | 功能 |
|------|------|
| `↑` / `↓` / `j` / `k` | 选择模块 |
| `Enter` | 进入选中的模块（同时持久化 last_module）|
| `Esc` | 关闭弹窗（仅在通过 Ctrl+P 打开时可用；首次启动 Esc 无效）|
| `q` | 退出程序（首次启动；二次打开时同样退出）|
| `Ctrl+C` | 强制退出 |

模块内：

| 按键 | 功能 |
|------|------|
| `Ctrl+P` | 重新打开入口弹窗，切换到其他模块 |

## 持久化

`runtime-config.yaml` 的 `last_module` 字段：

```yaml
first_run: false
last_module: skills   # skills | mcp | settings
```

- `Enter` 进入某模块后立刻写盘
- 字段缺失 / 取值非法 → 视为首次启动，强制弹窗
- 写盘失败不阻塞使用（下次启动会回到弹窗）

## 设计要点

- **首次启动强制弹窗**：让用户明确开始；不直接默认进 SKILLS 是因为我们希望用户至少知道 MCP / SETTINGS 的存在
- **后续启动跳过**：日常使用最爽，不用每次按 Enter
- **Ctrl+P 是唯一入口**：模块内导航只有 `Tab` 切 tab，没有"切模块"快捷键，避免按错误打断当前工作流
- **canClose=false 的弹窗**：第一次启动时 Esc 不响应（没有可回退的状态）；通过 Ctrl+P 打开时 Esc 可关回到原模块
